# M28.7 奇偶层模板与 WebSocket status 配置说明

## 1. 修改内容

### 奇偶层交错模板

`first_layer_placement` 现在支持两套标准模板：

- 奇数层：模板 `A` / 配置键 `odd`；
- 偶数层：模板 `B` / 配置键 `even`；
- 第 3、5、7...层自动回到模板 A；
- 第 4、6、8...层自动使用模板 B。

模板 B 将四个区域的横竖方向与覆盖位置相对模板 A 错开，使相邻两层交错咬合。
下一层不再直接复制上一层四个 OBB 检测框，而是始终在锁定托盘的居中正方形 footprint
上生成当前层标准模板，避免布局完全相同和误差逐层累积。

关键配置：

```yaml
task:
  algorithm:
    layering:
      next_layer_geometry: layer_template
      use_previous_detected_boxes: false

    template:
      layer_strategy: odd_even
      default_template: odd
      slot_order: [P3, P1, P2, P4]
      templates:
        odd:
          template_id: A
          slots: [...]
        even:
          template_id: B
          slots: [...]
```

业务结果会增加：

```json
{
  "template": {
    "strategy": "odd_even",
    "key": "even",
    "template_id": "B",
    "layer": 2,
    "geometry_source": "layer_template"
  }
}
```

每个 `slots[]` 也包含 `template_key` 和 `template_id`，仅用于调试和可视化，机器人
`detection.items[]` 协议不变。

### WebSocket status 推送

新增配置：

```yaml
task:
  communication:
    websocket:
      status_enabled: false
      status_on_connect: false
      status_interval_s: 1.0
```

- `status_enabled=false`：不启动周期 status 线程；
- `status_on_connect=false`：机器人刚连接时也不主动发 status；
- trigger 对应的 detection、原生 Ping/Pong 和 JSON ping/pong 不受影响。

需要恢复协议状态心跳时，将两项改为 `true`。

模拟机器人客户端默认隐藏 status；需要查看时：

```bash
python3 -m production.carton_palletizing.tasks.first_layer_placement.mock_robot_client \
  --url ws://127.0.0.1:9001/vision \
  --show-status
```

## 2. 已安装设备的配置迁移

systemd 通常读取：

```text
/etc/visionops_v3/carton_palletizing.yaml
```

安装脚本会保留这个文件，因此覆盖代码后需要把仓库中的以下配置合并进去：

```text
task.communication.websocket.status_enabled
task.communication.websocket.status_on_connect
task.algorithm.layering.next_layer_geometry
task.algorithm.layering.use_previous_detected_boxes
task.algorithm.template.layer_strategy
task.algorithm.template.default_template
task.algorithm.template.templates.odd
task.algorithm.template.templates.even
```

尚未进行现场参数定制时，可以备份并直接替换：

```bash
sudo cp /etc/visionops_v3/carton_palletizing.yaml \
  /etc/visionops_v3/carton_palletizing.yaml.bak
sudo cp /opt/visionops_v3/production/carton_palletizing/config/line.yaml \
  /etc/visionops_v3/carton_palletizing.yaml
```

然后重启：

```bash
sudo systemctl restart \
  visionops-v3-carton-palletizing-runtime.service \
  visionops-v3-carton-palletizing-app.service \
  visionops-v3-carton-palletizing-collector.service
```
