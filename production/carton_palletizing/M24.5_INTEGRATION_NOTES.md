# M24.5 纸箱托盘摆放第一阶段集成说明

本目录由 `visionops_v3_carton_palletizing_phase1_obb_fix` 的 OBB 第一层逻辑迁移到 M24.5 主线，并针对现场画面做了三项调整。

## 1. 生产画面完整放大

生产页面不再依赖图片固有尺寸。前端根据实时显示区域计算最大 `contain` 尺寸，显式设置图片宽高：

- 保持相机画面长宽比；
- 不裁剪任何边缘；
- 在显示区域内尽可能放大；
- Canvas 与实际图片元素严格对齐；
- 模型、耗时和 FPS 信息放到画面下方。

## 2. 第一层摆放顺序

默认顺序：

```text
P3（左下） -> P1（左上） -> P2（右上） -> P4（右下）
```

即从左下角开始，按画面中的顺时针方向摆放。

## 3. 居中正方形垛型

托盘 OBB 为长方形时，以短边作为正方形垛型边长：

- 竖向长托盘：左右占满，上下等距留白；
- 横向长托盘：上下占满，左右等距留白；
- 四块纸箱区域相对于这个正方形定义，不再随整个长方形托盘被拉伸。

关键配置：

```yaml
geometry:
  footprint_mode: centered_square_by_short_edge
  footprint_fill_ratio: 1.0

template:
  slot_order: [P3, P1, P2, P4]
```

## 已有 /etc 配置

安装脚本不会覆盖已有 `/etc/visionops_v3/carton_palletizing.yaml`。从旧版升级时，需手动合并或覆盖配置：

```bash
sudo cp /opt/visionops_v3/production/carton_palletizing/config/line.yaml \
  /etc/visionops_v3/carton_palletizing.yaml
```

覆盖前请备份已经现场标定过的参数。

## 4. 多层 RGB-D 状态机

本次在第一层基础上继续加入多层逻辑：

- 第一层仍使用 OBB 检测确认 P1～P4；
- 当前层完成后采集多帧稳定深度作为下一层基准；
- 第2层及以后通过 `baseline_depth - current_depth` 判断新增纸箱；
- 每层结束都更新基准，因此第3、4层不会一直与空托盘比较；
- 默认 `max_layers=4`，可改成任意正整数，设为0时不限层数；
- 下一层优先继承上一层实际纸箱 OBB 作为摆放掩膜。

从第一层版本升级时，必须把以下配置同步到 `/etc/visionops_v3/carton_palletizing.yaml`：

```yaml
camera_bridge:
  depth_path: /stream/depth.png
  max_depth_age_ms: 1500

task:
  algorithm:
    layering:
      max_layers: 4
      auto_advance: true
      baseline_capture_frames: 3
      baseline_settle_frames: 5
      baseline_stability_mm: 15.0
      use_previous_detected_boxes: true
    depth:
      min_height_delta_mm: 80.0
      max_height_delta_mm: 600.0
      min_coverage_ratio: 0.55
```
