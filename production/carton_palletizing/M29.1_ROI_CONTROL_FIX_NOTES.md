# M29.1 carton_palletizing ROI 控制权修复

## 问题

Runtime 已输出托盘/纸箱 OBB，但机器人连接后下发的 `config.detect_region` 会被 App 再次用于像素区域过滤。机器人使用 640×480 的区域，而相机实际为 1280×720 时，合法目标会在 App 二次筛选中被删除，表现为：

```text
Runtime detections 有 tray/box
surface_target_selection.candidate_*_count = 0
robot_message.items = []
```

## 修复

- ROI 只由 VisionOps Web 页面和 Runtime 的 `roi_config_path` 控制；
- Runtime 输出的 `detections[]` 已是 Web ROI 内结果，App 不再进行机器人 ROI 二次过滤；
- WebSocket `config.detect_region` 仍兼容接收，但只记录，不应用；
- `pallet_place_target` 与 `held_box_pose` 都遵循相同规则；
- `confidence_threshold` 默认仍允许机器人动态调整，可用 `remote_config.allow_confidence_threshold=false` 关闭；
- 诊断增加原始候选数、置信度过滤后候选数、忽略的机器人 ROI 和 ROI 控制来源。

## 诊断字段

```json
{
  "roi_control_source": "visionops_web_runtime",
  "robot_detect_region_applied": false,
  "last_ignored_robot_detect_region": [100.0, 10.0, 540.0, 470.0],
  "raw_candidate_tray_count": 1,
  "confidence_filtered_tray_count": 1
}
```

## 部署后验证

机器人连接并下发任意 `detect_region` 后，执行：

```bash
curl -s -X POST http://127.0.0.1:19210/api/app/trigger \
  -H 'Content-Type: application/json' \
  -d '{"task_id":"pallet_place_target"}' | python3 -m json.tool
```

只要 Runtime 的 `detections[]` 中存在 Web ROI 内托盘/纸箱，`robot_message.items` 就不会再因机器人 ROI 而变空。
