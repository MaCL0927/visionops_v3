# M25.3 抓取点协议 v1.2 修改说明

基线：`visionops_v3_M25.3_box_grasp_protocol_section9_v1.1`

## 修改内容

1. `box_grasp_vision` 的机器人报文由“一个纸箱一个 items 元素、内部包含左右字段”改为“一个抓取点一个 items 元素”。
2. 同一个纸箱的两个抓取点使用相同的 `id/class_id/confidence`，分别携带各自的：
   - `position_camera`
   - `center_px`
3. 抓取点不再使用 `left_grasp_px/right_grasp_px/left_grasp_camera/right_grasp_camera` 字段。
4. 同一纸箱的两个点按像素 x 坐标升序输出，但机器人应按 `id` 分组。
5. `box_grasp_vision` WebSocket 默认端口统一改为 `9001`，不同任务通过视觉盒 IP 区分。
6. `tube_pick_vision` 代码与既有协议未修改；其 items 原本就是一个产品一个抓取点，字段已与新结构一致。
7. 分割轮廓、四角点、纸箱中心和深度调试数据继续保留在 `visualization_result`，不进入机器人抓取点报文。
