# VisionOps v3 AI Agent 协作规则

本文适用于仓库内所有 AI agent 和自动化编码工具。任何子目录新增的 `AGENTS.md` 只能增加更具体的约束，不得削弱本文规则。

## 1. 基本原则

1. 修改前先阅读相关架构文档、接口定义、配置说明和现有测试。
2. v3 是重新设计的项目，不得整目录复制 v2，也不得为了快速兼容引入旧结构。
3. 每次修改保持单一目的，避免顺手重构无关模块。
4. 优先建立明确的数据结构、进程边界和失败语义，再实现业务逻辑。
5. 不得把临时方案描述为生产方案；未实现能力必须明确标记状态。

## 2. 主链路约束

1. 生产边缘主链路固定为：`Camera Bridge -> C++ RKNN Runtime -> Collector Web -> Gateway/Modbus`。
2. RKNN 模型加载、预处理、NPU 推理和生产后处理默认使用 C++ 实现。
3. Python 主要用于 Web、配置、训练、导出、部署编排和诊断，不得新增 Python 生产 RKNN 主链路。
4. Python 相机链路、Python RKNN 链路和 ROS1 bridge 仅可记录在 `docs/legacy_notes/`，不得成为默认依赖或启动路径。
5. RK3588 与 LB3576 的硬件差异必须封装在设备适配层或配置中，不得散落在业务逻辑里。

## 3. 模块边界

1. `apps/` 不直接实现 NPU 推理，只通过稳定接口调用边缘运行时。
2. `training/` 不依赖边缘设备本地路径；模型发布通过版本化制品契约完成。
3. `edge/` 中各进程不得读取其他模块的内部文件作为隐式 API。
4. 跨进程、跨语言数据结构必须定义在 `interfaces/`，并包含版本字段。
5. Gateway 与 Modbus 适配器消费标准化检测结果，不得绑定具体模型输出张量。
6. `edge/common/` 只放稳定且被多个边缘模块复用的能力，禁止形成无边界工具箱。

## 4. 配置规则

1. 人工维护的配置分为 `configs/edge/`、`configs/task/` 和 `configs/app/`。
2. `configs/runtime/` 只保存生成产物的说明或示例；真实运行时环境文件不得提交。
3. 生成文件必须可由源配置稳定重建，并标注来源、生成时间和配置版本。
4. 代码不得依赖开发者机器的绝对路径、用户名、固定 IP 或 Conda 环境路径。
5. 密钥、密码、Token 和设备私有信息只能通过受控环境变量或外部密钥系统注入。
6. 新增配置项时必须同时更新配置文档、校验规则和默认行为说明。

## 5. 安全与仓库卫生

1. 禁止提交 `.env`、私钥、证书私钥、访问凭据和生产数据库快照。
2. 禁止提交 `.pt`、`.pth`、`.onnx`、`.rknn`、模型缓存和训练输出。
3. 禁止提交数据集、采集图片、视频、日志、诊断包和压缩包。
4. 不得绕过 `.gitignore` 强制添加上述文件。
5. 示例配置必须使用明显的占位值，不得包含可用的生产凭据。

## 6. 实现与依赖

1. C++ 代码优先使用 CMake 组织，明确目标平台、编译选项和第三方依赖。
2. Python 包应有清晰的包边界和锁定策略，不得依赖全局解释器状态。
3. 新依赖必须说明用途、许可证、目标架构支持情况和部署成本。
4. 第三方 SDK 不直接复制进仓库；优先记录版本、校验值和获取方式。
5. 日志应结构化并携带模块、请求或帧标识，禁止记录密钥和大块图像数据。

## 7. 测试与验证

1. 新功能必须提供与风险匹配的单元测试或集成测试。
2. 跨模块接口变更必须增加契约测试，并说明兼容性影响。
3. 硬件相关代码应尽可能将纯逻辑与设备调用分离，以便在非 ARM 环境测试。
4. 不能执行硬件验证时，必须明确说明未验证项，不能以模拟结果代替真机结论。
5. 修改部署、systemd 或配置生成逻辑时，至少验证语法、路径和幂等性。

## 8. 文档与迁移

1. 所有项目文档使用中文；协议字段和代码标识可保留英文。
2. 架构决策变化必须同步更新 `docs/architecture/`。
3. 从 v2 迁移前必须先写迁移清单，说明保留的能力、放弃的实现和验收方式。
4. 迁移应重新实现最小必要能力；不得携带无调用方代码、废弃脚本、历史模型或本机配置。
5. 历史方案仅记录在 `docs/legacy_notes/`，不得与当前架构文档混写。

## 9. 完成标准

一次修改只有在以下条件满足时才算完成：职责边界清楚，配置和文档同步，测试或验证完成，没有引入敏感文件或大型制品，并且未破坏 C++ RKNN 生产主链路。
