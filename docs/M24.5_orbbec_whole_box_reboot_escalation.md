# M24.5 Orbbec 336L 连续恢复失败后的整机重启

## 目标

在 M24.3 Bridge 自动重连、systemd watchdog 兜底和 M24.4 简化机器人故障码基础上，
增加最后一级恢复措施：相机持续异常且 Bridge 服务连续重启 10 次仍没有恢复 RGB/Depth
新帧时，自动重启整个 RK3576 视觉盒。

## 恢复顺序

```text
相机 RGB/Depth 停帧
  → Bridge 内部重建 Orbbec Pipeline
  → 相机持续异常超过 30 秒
  → watchdog 重启 visionops-orbbec336l-bridge.service
  → 等待 RGB/Depth 恢复
  → 未恢复：失败次数 +1
  → 连续失败达到 10 次
  → systemctl reboot --no-block
```

watchdog 以 root 身份运行，`systemctl reboot` 等价于人工执行 `sudo reboot`。

## 防止循环重启

失败次数和本次故障是否已执行 reboot 保存在：

```text
/var/lib/visionops_v3/watchdog/
├── orbbec336l_failed_service_restarts
├── orbbec336l_incident_started_at
└── orbbec336l_reboot_issued
```

默认同一相机故障事件只允许执行一次整机重启。视觉盒重启后，如果 USB 相机仍未插入，
不会再次循环 reboot。只要 RGB 和 Depth 都恢复为新鲜帧，以上计数和标记会自动清零，
下一次独立故障事件可以重新使用整机重启恢复。

## 配置

配置文件：

```text
/etc/visionops_v3/carton_line.env
```

默认值：

```bash
VISIONOPS_CAMERA_WATCHDOG_RESTART_WHILE_UNHEALTHY=true
VISIONOPS_CAMERA_WATCHDOG_UNHEALTHY_RESTART_AFTER_S=30
VISIONOPS_CAMERA_WATCHDOG_MAX_SERVICE_RESTARTS=10
VISIONOPS_CAMERA_WATCHDOG_REBOOT_ENABLED=true
VISIONOPS_CAMERA_WATCHDOG_REBOOT_DELAY_S=5
VISIONOPS_CAMERA_WATCHDOG_REBOOT_ONCE_PER_INCIDENT=true
VISIONOPS_CAMERA_WATCHDOG_PERSIST_DIR=/var/lib/visionops_v3/watchdog
```

相关的服务重启间隔仍由以下参数控制：

```bash
VISIONOPS_CAMERA_WATCHDOG_COOLDOWN_S=60
VISIONOPS_CAMERA_WATCHDOG_RECOVERY_WAIT_S=20
```

默认情况下，连续 10 次失败通常需要约 10 分钟，不会在短时间内频繁重启相机服务。

## 安全操作

计划拔相机、维护 USB 线或测试断线时，应先暂停 watchdog timer，避免测试过程中达到整机重启阈值：

```bash
sudo systemctl stop visionops-orbbec336l-bridge-watchdog.timer
```

测试完成后恢复：

```bash
sudo systemctl enable --now visionops-orbbec336l-bridge-watchdog.timer
```

临时禁止整机重启但保留 Bridge 服务自动恢复：

```bash
sudo sed -i \
  's/^VISIONOPS_CAMERA_WATCHDOG_REBOOT_ENABLED=.*/VISIONOPS_CAMERA_WATCHDOG_REBOOT_ENABLED=false/' \
  /etc/visionops_v3/carton_line.env
sudo systemctl restart visionops-orbbec336l-bridge-watchdog.timer
```

手工清零故障升级计数：

```bash
sudo rm -f \
  /var/lib/visionops_v3/watchdog/orbbec336l_failed_service_restarts \
  /var/lib/visionops_v3/watchdog/orbbec336l_incident_started_at \
  /var/lib/visionops_v3/watchdog/orbbec336l_reboot_issued
```

下一次 watchdog 运行会自动重新创建计数文件。

## 查看状态

```bash
sudo cat /var/lib/visionops_v3/watchdog/orbbec336l_failed_service_restarts
sudo cat /var/lib/visionops_v3/watchdog/orbbec336l_reboot_issued 2>/dev/null || true
sudo journalctl -t visionops-orbbec-watchdog -n 100 --no-pager
systemctl status visionops-orbbec336l-bridge-watchdog.timer --no-pager
```

达到阈值前日志类似：

```text
Bridge 服务重启后相机仍不可用；failed_service_restarts=9/10
```

达到阈值时：

```text
CRITICAL: visionops-orbbec336l-bridge.service 连续重启 10 次后相机仍异常；5s 后重启视觉盒子
```
