# M25.3 边缘端环境依赖审计与部署说明

## 结论

当前 VisionOps v3 边缘端 Python 代码直接依赖：

- `numpy`、`cv2`：图像、深度和几何处理；
- `PyYAML`：配置读取；
- `requests`：HTTP 调用；
- `psutil`：系统状态；
- `paramiko`：仅用于数据包 SSH 上传，缺失时可回退到系统 `ssh/scp`；
- `packaging`、`typing_extensions`：Python 3.8 兼容辅助。

当前 v3 边缘端不再使用 v2 Python 推理服务，因此不需要把以下 v2 依赖照搬进新 venv：

- `fastapi`、`uvicorn`、`python-multipart`；
- `pymodbus`；
- `rknn-toolkit-lite2`；
- pip 版 `opencv-python-headless`。

v3 的推理服务为 C++ RKNN Runtime，依赖板端的 `librknnrt.so`、`rknn_api.h` 和可选的 `librga.so`。这些不是 Python venv 包。

## 本次验证失败的原因

旧版 `setup_edge_env.sh` 验证代码尝试导入：

```python
from production.carton_line.gateway.config import load_line_config
```

M25.3 中真实公开函数名是：

```python
from production.carton_line.gateway.config import load_config
```

因此失败发生在“仓库符号名写错”，不是 Python 依赖缺失。此前 pip 安装已经成功完成，venv 本身可继续使用。

## launchpadlib 警告

Ubuntu 系统环境中的 `launchpadlib` 声明依赖 `testresources`。由于 venv 使用 `--system-site-packages` 复用板端 OpenCV，pip 能看到该系统包，从而提示元数据不完整。

`launchpadlib` 并非 VisionOps 依赖，该提示不影响 VisionOps。修订后的安装脚本会安装 `python3-testresources`，避免再次出现该警告。

## 验证

修订脚本支持只验证，不重复安装：

```bash
sudo bash /opt/visionops_v3/scripts/setup_edge_env.sh --verify-only
```

完整重建：

```bash
sudo bash /opt/visionops_v3/scripts/setup_edge_env.sh --recreate --with-dev
```
