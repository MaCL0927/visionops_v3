# M24.3 Orbbec 336L 断线重连与相机报警

## 目标

用于 `tube_pick_vision` 7×24 小时无人值守运行。USB 相机拔出、RGB/Depth 停帧或
Orbbec SDK 卡住时，系统不再持续返回最后一帧，而是自动恢复并向机器人输出稳定的相机故障状态。

## 恢复链路

```text
RGB/Depth 超过 3 秒无新帧
  → Bridge camera_state=stale，清空旧缓存，snapshot/depth 返回 503，MJPEG 断流
  → Bridge 完整停止并销毁旧 Pipeline/设备句柄
  → 重新枚举 336L、选择 640×480@30、恢复 D2C、重新读取标定参数
  → 1/2/4/8…30 秒指数退避重试
  → RGB+Depth 首帧均有效后 camera_state=running
  → Runtime 自重连；外部 watchdog 必要时重置 preview/重启 Runtime
```

如果 SDK 卡在 `waitForFrames()` 或 `pipeline->stop()`，Bridge 内部线程无法继续，独立
`visionops-orbbec336l-bridge-watchdog.timer` 会根据恢复线程与重连计数是否长期无进展来重启 Bridge。
物理相机持续未插入时，内部重连计数仍在推进，watchdog 不会周期性制造重启风暴。

## Bridge `/health` 新字段

- `camera_connected`
- `camera_state`: `starting/running/stale/reconnecting/offline/stopping`
- `camera_thread_alive`
- `frame_stale`
- `fault_code` / `fault_numeric_code`
- `alarm_active`
- `last_color_age_ms` / `last_depth_age_ms`
- `reconnect_attempt_count` / `reconnect_success_count`
- `consecutive_reconnect_failures`
- `stale_event_count`
- `unhealthy_age_ms`

## tube_pick_vision 行为

相机不健康时，在调用 Runtime 前即终止本次推理，防止旧缓存进入业务：

```json
{
  "type": "detection",
  "request_id": "pick-1",
  "items": [],
  "camera_connected": false,
  "camera_state": "reconnecting",
  "error": {"code": "CAMERA_DISCONNECTED", "message": "..."},
  "alarm": {
    "active": true,
    "confirmed": true,
    "code": "CAMERA_RECONNECTING",
    "numeric_code": 3102,
    "severity": "error",
    "modbus_tcp_reserved": true,
    "modbus_tcp_implemented": false
  }
}
```

## Modbus-TCP 预留

`pick.camera_health.modbus_tcp` 当前默认：

```yaml
modbus_tcp:
  enabled: false
  reserved: true
  fault_register: null
  state_register: null
```

当前不监听或写入任何 PLC 寄存器。后续 Modbus-TCP 适配器可直接消费 WebSocket/HTTP
状态中的字符串故障码和数值故障码：

- `0`: 正常
- `3100`: CAMERA_STARTING
- `3101`: CAMERA_FRAME_STALE
- `3102`: CAMERA_RECONNECTING
- `3103`: CAMERA_OFFLINE
- `3104`: CAMERA_BRIDGE_UNREACHABLE

## 安装

```bash
cd /opt/visionops_v3

sudo bash edge/camera_bridge/orbbec336l_bridge/install_orbbec336l_bridge_service.sh
sudo bash production/carton_line/deploy/install_services.sh --profile tube-pick

sudo systemctl restart visionops-orbbec336l-bridge.service
sudo systemctl restart visionops-v3-runtime-pick.service visionops-v3-ws-pick.service
sudo systemctl enable --now visionops-orbbec336l-bridge-watchdog.timer
sudo systemctl enable --now visionops-v3-runtime-pick-watchdog.timer
```

## 检查

```bash
curl -s http://127.0.0.1:18182/health | python3 -m json.tool
curl -s http://127.0.0.1:19130/api/app/status | python3 -m json.tool
systemctl status visionops-orbbec336l-bridge-watchdog.timer
systemctl list-timers --all | grep visionops
journalctl -t visionops-orbbec-watchdog -f
```

## 拔插测试

1. 正常运行时确认 `camera_connected=true`；
2. 拔掉 USB，约 3 秒后确认 `camera_state` 进入 `stale/reconnecting/offline`；
3. `snapshot.jpg` 应返回 HTTP 503，MJPEG 客户端应断开；
4. WebSocket 应收到 `camera_connected=false`、空 `items` 和相机告警；
5. 重新插入 USB；
6. 等待指数退避周期，确认 Bridge 自动回到 `running`；
7. 确认 Runtime 画面与推理自动恢复，无需人工 `systemctl restart`。
